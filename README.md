# FocusManager PWA v1.2.2 Sticky Clock

Session 스타일의 개인 집중 관리 PWA입니다.  
핵심 흐름은 **의도 설정 → 집중 세션 실행 → 회고 기록 → 타임라인 정리 → 리포트 분석 → 백업**입니다.

손해사정사 공부, 장시간 학습, 업무 집중, 하루 시간 사용 점검을 목적으로 합니다.

## 현재 버전 핵심

v1.2.2는 v1.2.1 Modal Hotfix를 기준으로 화면 상단에 항상 보이는 작은 현재시간 배지를 추가한 사용성 개선 버전입니다.

GitHub 백업 기반과 모바일 모달 안정화는 유지하면서, 긴 페이지를 스크롤해도 현재 시간을 바로 확인할 수 있도록 고정 시계 UI를 추가했습니다.

- 기존 localStorage 저장 유지
- 기존 JSON 백업/복원 유지
- GitHub 백업 설정 추가
- GitHub 수동 백업 추가
- GitHub 백업 복원 추가
- 세션 종료 후 자동 백업 옵션 추가
- GitHub 토큰은 백업 파일에 포함하지 않음
- 저장 키 `focus_manager_v1` 유지
- 수정 모달 hidden 표시 오류 수정
- 모바일 수정 모달 스크롤/닫기 UX 안정화
- 화면 상단 고정 현재시간 표시

## GitHub 백업 사용 방법

설정 화면의 GitHub 백업 영역에 다음 값을 입력합니다.

- Owner: GitHub 아이디 또는 조직명
- Repo: 백업 파일을 저장할 저장소명
- Branch: 보통 `main`
- Backup path: 기본값 `data/focusmanager_backup.json`
- GitHub token: Contents 읽기/쓰기 권한이 있는 토큰

그다음 `GitHub 백업 사용`을 켜고 `GitHub 백업 실행`을 누르면 백업 JSON이 저장소에 커밋됩니다.

세션 종료 후 자동 백업을 원하면 `세션 종료 후 자동 백업`도 켭니다.

## 백업 데이터 기준

GitHub 백업 파일은 다음 형식의 envelope 구조로 저장됩니다.

```json
{
  "app": "FocusManager",
  "backupVersion": "1.0",
  "version": "1.2.2",
  "storageKey": "focus_manager_v1",
  "backupAt": "ISO_DATE",
  "data": {
    "settings": {},
    "categories": [],
    "sessions": [],
    "activeSession": null
  }
}
```

중요: GitHub token은 `data.settings.githubBackup.token`에 저장하지 않고 빈 문자열로 백업됩니다.  
토큰은 각 기기의 localStorage에만 남습니다.

## 데이터 기준

- 저장소: `localStorage`
- 저장 키: `focus_manager_v1`
- 기존 v1.0/v1.1 데이터와 호환되도록 저장 키는 변경하지 않았습니다.
- 기존 JSON 백업 파일과 v1.2 envelope 백업 파일 모두 가져올 수 있습니다.

## 현재 제한

- 여러 기기 실시간 동기화는 아직 없습니다.
- GitHub 백업은 수동 복원 중심입니다.
- 충돌 병합 기능은 아직 없습니다.
- 여러 기기에서 동시에 세션을 기록하면 나중에 복원 시 덮어쓰기 충돌이 생길 수 있습니다.
- PWA 단독으로는 앱/웹사이트 차단, Slack 제어, Mac 시스템 제어가 어렵습니다.
- 해당 기능은 후속 버전에서 Mac 앱 또는 브라우저 확장 기능으로 분리 검토합니다.
